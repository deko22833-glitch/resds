# Мессенджер

Полнофункциональный мессенджер с регистрацией, системой друзей и видеозвонками, работающий полностью на клиентской стороне.

## Возможности

- ✅ Регистрация новых пользователей (без почты)
- ✅ Вход в систему
- ✅ Система друзей (добавление и удаление)
- ✅ Приватные чаты с друзьями
- ✅ Видеозвонки через WebRTC
- ✅ Голосовые звонки
- ✅ Управление видео/аудио во время звонка
- ✅ Современный и красивый интерфейс
- ✅ Адаптивный дизайн

## Установка и запуск

1. Клонируйте репозиторий или скачайте файлы
2. Откройте `index.html` в браузере
3. Зарегистрируйтесь или войдите в систему
4. Начните общаться!

## Настройка Firebase (для работы с разных компьютеров)

Для того, чтобы пользователи с разных компьютеров могли регистрироваться и общаться, необходимо настроить Firebase:

1. Перейдите на https://console.firebase.google.com/
2. Создайте новый проект (или выберите существующий)
3. В настройках проекта найдите раздел "Ваши приложения" → нажмите на иконку Web `</>`
4. Зарегистрируйте приложение (можно указать любое имя)
5. Скопируйте конфигурацию Firebase (она будет выглядеть примерно так):
   ```javascript
   const firebaseConfig = {
     apiKey: "AIza...",
     authDomain: "your-project.firebaseapp.com",
     databaseURL: "https://your-project-default-rtdb.firebaseio.com",
     projectId: "your-project",
     storageBucket: "your-project.appspot.com",
     messagingSenderId: "123456789",
     appId: "1:123456789:web:abc123"
   };
   ```
6. Откройте файл `firebase-config.js` и вставьте вашу конфигурацию
7. В Firebase Console перейдите в "Realtime Database" → "Создать базу данных"
8. Выберите режим "Режим тестирования" (для начала)
9. Выберите регион (ближайший к вам)
10. После создания базы данных, перейдите в "Правила" и установите следующие правила для тестирования:
    ```json
    {
      "rules": {
        ".read": true,
        ".write": true
      }
    }
    ```
    **Внимание:** Эти правила позволяют всем читать и писать данные. Для продакшена настройте правила безопасности!
11. Готово! Теперь все пользователи будут видеть друг друга

**Важно:** 
- Если Firebase не настроен, приложение будет работать в локальном режиме (только на одном компьютере)
- Для продакшена обязательно настройте правила безопасности Firebase в разделе "Правила" базы данных

## Развертывание на GitHub Pages

1. Создайте новый репозиторий на GitHub
2. Загрузите все файлы в репозиторий
3. Перейдите в Settings → Pages
4. Выберите ветку `main` и папку `/ (root)`
5. Сохраните изменения
6. Ваш сайт будет доступен по адресу: `https://ваш-username.github.io/имя-репозитория/`

## Технологии

- HTML5
- CSS3 (с градиентами и анимациями)
- Vanilla JavaScript
- Firebase Realtime Database (для синхронизации между устройствами)
- LocalStorage (для офлайн режима и fallback)
- WebRTC (для видеозвонков)

## Примечание

**Синхронизация данных:** 
- Если Firebase настроен: все данные (пользователи, сообщения, друзья) синхронизируются в реальном времени между всеми устройствами через Firebase Realtime Database
- Если Firebase не настроен: данные хранятся только в LocalStorage браузера и доступны только на одном компьютере

**Звонки:** Видео и голосовые звонки работают через WebRTC. Для полноценной работы между разными устройствами рекомендуется использовать signaling сервер. В текущей реализации звонки работают через localStorage как механизм signaling, что подходит для демонстрации и локального использования.
